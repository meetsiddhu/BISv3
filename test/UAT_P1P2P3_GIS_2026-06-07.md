## 4. Findings & decisions

- **[F-1 / P1.1] ALP bar chart does not render bars — Low.** A first fix marked the
  `restrUnit` measure `@Analytics.Measure`, which **flipped the shared report entity into
  analytical mode and broke the flat ALV list** (caught in this UAT). Reverted to the
  dynamic `AggregatedProperty` form → ALV restored (6 rows render). The bar chart remains
  a documented FE-ALP limitation; **slice-and-dice is fully delivered** via the 7-filter
  bar + flat ALV + backend `groupby/sum` aggregation. *No data impact.*
- **[F-2 / P2.5 / C-3] draft-activate 404 — environmental, not a code defect.** During the
  UI save, the BTP **trial auto-stopped the srv + approuter** (`requested state: stopped`),
  surfacing a CF router `404 route does not exist`. Recovered with
  `cf start BridgeManagement-srv && cf start BridgeManagement`; the override change then
  activated cleanly (`200`, 40.00/High). Recommendation: keep HANA/app awake during demos
  (runbook covers the start step).
- **[F-3] FE model staleness after out-of-band change — cosmetic.** Editing via API while
  an object page was already open showed stale values until a fresh model load; expected
  behaviour, not a defect.
- **[P3.8] Live S/4 EAM integration — deferred (external dependency).** Needs a reachable
  S/4 system + BTP destination + user credentials; the app stays standalone/S/4-compatible
  (`EAMCodeMapping` + reference fields in place). Cannot be exercised without S/4.
- **[P3.9] Native HANA `ST_GEOMETRY` — superseded.** The portable GeoJSON `LargeString`
  approach delivers Point/Line/Polygon/Polyline in **both** SQLite dev and HANA without
  breaking the dev/test path — the better engineering choice.
